# Restaurant-menu-angular

## Описание

Небольшое SPA-приложение на Angular с маршрутизацией, standalone-компонентами и локальным управлением состоянием.  
Проект демонстрирует базовые frontend-навыки: работу с компонентной архитектурой, роутингом, реактивным состоянием и построением интерфейса без лишней сложности.

## О проекте

Приложение состоит из двух экранов:

- **Home** — стартовая страница;
- **Menu** — страница с разделами меню и выбором позиций.

На странице меню пользователь может переключаться между категориями, отмечать нужные позиции и сразу видеть итог: количество выбранных элементов и общую сумму заказа.

Проект сделан как компактное, аккуратное Angular-приложение с упором на чистую структуру, понятную логику и разделение ответственности.

## Что реализовано

- маршрутизация между страницами;
- standalone-компоненты;
- отдельный сервис для хранения состояния;
- использование `signal` и `computed`;
- отображение категорий меню;
- выбор и снятие выбора у позиций;
- автоматический подсчёт количества выбранных элементов;
- автоматический подсчёт общей суммы;
- раздельное хранение логики, данных и UI.

## Логика работы

Все данные хранятся локально в статическом файле.

### Разделы меню

Каждый раздел содержит:

- `id`
- `title`
- список элементов `items`

### Элемент меню

Каждый элемент содержит:

- `id`
- `title`
- `value`
- `selected`

При выборе раздела справа отображаются только его элементы.

При переключении `checkbox`:

- обновляется количество выбранных пунктов;
- пересчитывается сумма значений;
- данные меняются только в рамках текущего раздела.

## Маршруты

- `/` — главная страница
- `/menu` — страница меню
